Duplicate Detection — 3 Layers

How the system prevents duplicate invoices across Email and Manual Upload channels

Layer 1: Email ID
Gmail message dedup
How it works

Every Gmail email has a unique messageId. We store this when an email is processed. If the same email appears again in a future poll, it's silently skipped.

Prevents: Same email processed twice across polling cycles
Action: Silently skipped — no user intervention
Layer 2: File Hash
SHA-256 content fingerprint
How it works

Every uploaded/downloaded file is hashed (SHA-256). Before processing, we check if the exact same file content already exists. Catches the same PDF uploaded manually AND arriving via email.

Prevents: Same file uploaded via two channels (email + manual)
Action: Warning shown — user can keep or discard
Layer 3: Invoice Data
OCR-extracted fields match
How it works

After OCR, we check if supplier + invoice # + date match an existing record. Catches re-sent invoices (same data, different PDF) and corrected invoices.

Prevents: Re-sent or corrected invoices creating duplicates
Action: Flagged for review — user decides (keep/discard/replace)
