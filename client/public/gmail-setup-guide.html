Connect Gmail to RetailEdge

Follow these steps to create Google Cloud OAuth credentials and connect your Gmail account for automatic invoice imports.

1
Create a Google Cloud Project
If you already have a project, skip to Step 2

Go to the Google Cloud Console and sign in with your Google account.

3

Before creating credentials, you need to set up the OAuth consent screen.

OAuth consent screen

gmail.readonly Read-only access to Gmail
userinfo.email See your email address
  1. Go to APIs & Services → OAuth consent screen
  2. If prompted, choose External user type (or Internal if using Google Workspace)
  3. Click Get Started and fill in the form:
    • App name: RetailEdge
    • User support email: your email
  4. Click Add or Remove Scopes and add:
    • https://www.googleapis.com/auth/gmail.readonly
    • https://www.googleapis.com/auth/userinfo.email
  5. Under Test Users, add the Gmail address you want to connect
  6. Click Save and Continue
⚠
Important: You must add your Gmail address as a Test User, or you won't be able to authorize the connection. Your app stays in "Testing" mode until you publish it.
4
Create OAuth 2.0 Credentials

Now create the OAuth credentials that RetailEdge will use to connect to Gmail.

Credentials

+ CREATE CREDENTIALS
API key
OAuth client ID
Service account
  1. Go to APIs & Services → Credentials
  2. Click Create Credentials (highlighted above)
  3. Select OAuth client ID from the dropdown
  4. Set application type to Web application
  5. Name it "RetailEdge"
  6. Under Authorized redirect URIs, click Add URI and paste this exact URL:
http://localhost:3001/api/gmail/oauth/callback
ℹ
The redirect URI must match exactly. If your RetailEdge server runs on a different port or domain, update accordingly.
  1. Click Create

Troubleshooting
Common issues and how to fix them
"redirect_uri_mismatch" The redirect URI in Google Console must exactly match http://localhost:3001/api/gmail/oauth/callback
"access_denied" Make sure you added your Gmail address as a Test User in the OAuth consent screen (Step 3)
Popup blocked Allow popups for your RetailEdge URL in your browser settings
"invalid_grant" Your Gmail access was revoked. Click Disconnect in RetailEdge and reconnect
Lost your Client Secret Go back to Google Cloud Console → Credentials, delete the old OAuth client, and create a new one
