Design Notes

Multi-select (checkboxes)

User can select multiple products — e.g. POS and Shopify versions of the same item. Selected rows highlighted in teal.

Confidence Score

Percentage badge per row: green (80%+), amber (50-79%), red (<50%). Uses the containment algorithm which handles extra pack-size words.

Source Badge

Shows where product came from (POS, Shopify, etc.) in color-coded pill. Hyphens in Shopify handles normalized for search.

Pricing Columns

  • Curr. Cost — existing costPrice from product/variant
  • Sell Price — current selling price
  • Curr. Margin — (sellPrice - cost) / sellPrice as % and $
  • New Cost — from invoice base unit cost (highlighted if increasing)
  • New Margin — recalculated with new cost, same sell price
  • Change — percentage increase/decrease in cost (red = up, green = down)

Layout Change

Switched from 2-panel (left suggestions + right search) to table layout. Invoice context + search on the left (narrow), product table on the right (wide). The table format makes pricing comparisons much easier to scan.

Should AI (Claude) be used for matching?

Current approach: Tokenization + containment scoring is fast, free, and works well for exact/near-exact product names. It handles Shopify hyphens, extra pack sizes, and case differences.

Where AI would help:

  • Abbreviations: "Org Raw Cacao Pwdr" vs "Organic Raw Cacao Powder" (fuzzy can't match abbreviations)
  • Synonym handling: "Bicarb Soda" vs "Baking Soda" / "Capsicum" vs "Bell Pepper"
  • Supplier-specific naming: "MNC Cacao Premium 3kg" where "MNC" = Melbourne Nut Co brand prefix
  • Brand vs product: Understanding that "Callebaut Dark Couverture" maps to "Dark Chocolate Couverture"

Recommended hybrid approach: Run fuzzy matching first (instant, free). If confidence is below 80%, fall back to a Claude API call with the invoice line + top 10 candidate products. This gives best of both: fast for obvious matches, AI for ambiguous ones. Cost: ~$0.003 per ambiguous line.
