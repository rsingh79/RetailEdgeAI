Setup Guide

Connect Google Drive to RetailEdge AI

Create your own Google Cloud credentials and connect your Drive

⏱ 5–10 minutes

To connect Google Drive, you need to create an OAuth 2.0 Client in your own Google Cloud Console. This gives you full control over your credentials and which permissions are granted.

Already set up for Gmail? If you created a Google Cloud project for the Gmail integration, you can reuse the same project — just enable the Google Drive API and add the Drive redirect URI. Skip to Step 2.
1
Create a Google Cloud Project
If you already have a project, skip this step
  1. Go to console.cloud.google.com
  2. Click Select a project at the top, then New Project
  3. Enter a project name (e.g., RetailEdge) and click Create
  4. Make sure your new project is selected in the top bar

  3. Click on it and press the Enable button

3
Configure the OAuth Consent Screen
Tell Google what your app needs access to
  1. Go to APIs & Services → OAuth consent screen
  2. Choose External user type (unless you have Google Workspace)
  3. Fill in the required fields:
    • App name: RetailEdge
    • User support email: your email
    • Developer contact: your email
  4. On the Scopes page, click Add or Remove Scopes and add:
    • https://www.googleapis.com/auth/drive.readonly
    • https://www.googleapis.com/auth/userinfo.email
  5. On the Test users page, add your Google email address
  6. Click Save and Continue
Already configured for Gmail? If you set up the OAuth consent screen before, just make sure the drive.readonly scope is added alongside your existing scopes.
4
Create OAuth 2.0 Credentials
Generate your Client ID and Secret
  1. Go to APIs & Services → Credentials
  2. Click + Create Credentials → OAuth client ID
  3. Set Application type to Web application
  4. Enter a name (e.g., RetailEdge Drive)
  5. Under Authorized redirect URIs, click + Add URI and paste:
http://localhost:3001/api/drive/oauth/callback
Important: The redirect URI must match exactly, including the protocol (http vs https) and port number. No trailing slash.
  1. Click Create

Copy Your Client ID and Client Secret
Save these before closing the dialog

Copy the Client Secret now! You won't be able to see it again after closing this dialog. If you lose it, you'll need to create a new credential.
6
Paste Credentials into RetailEdge
Final step — connect your Drive
  1. In RetailEdge, go to Settings → Integrations → Google Drive
  2. Paste your Client ID and Client Secret into the form
  3. Click Save Credentials
  4. Click Connect Google Drive — a Google sign-in popup will appear
  5. Sign in and click Allow to grant read-only access
  6. Browse your Drive and select a folder to watch for invoices
All done! RetailEdge will now automatically check this folder for new invoice files. You can add more folders or change settings in Settings → Integrations.

Supported File Types

RetailEdge detects and processes these file types from your Drive folder:

  • ✓ PDF documents (.pdf)
  • ✓ JPEG images (.jpg, .jpeg)
  • ✓ PNG images (.png)
  • ✓ WebP images (.webp)

Files up to 20 MB. Other file types are ignored.

Troubleshooting

redirect_uri_mismatch
The redirect URI in Google Cloud must match exactly: http://localhost:3001/api/drive/oauth/callback. Check for typos, missing port, or wrong protocol (http vs https).
access_denied
Your Google account must be added as a Test User in the OAuth consent screen settings (Step 3 above).
Popup blocked
Allow popups from RetailEdge in your browser settings, then try again.
invalid_grant
The refresh token has expired or was revoked. Disconnect and reconnect Google Drive in RetailEdge settings.
Lost the Client Secret?
You cannot recover it. Go to Google Cloud Console → Credentials, delete the old credential, and create a new one. Then update the credentials in RetailEdge.
Can I reuse my Gmail project?
Yes! Use the same Google Cloud project. Just enable the Google Drive API (Step 2) and add the Drive redirect URI to your existing OAuth credential (Step 4).
